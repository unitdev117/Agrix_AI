# Agrix: A Multilingual AI-Driven Communication Platform for Agricultural Support
## [Executive Summary & Academic Project Blueprint]

### 1. Abstract
Agrix is an advanced communication ecosystem designed to bridge the digital and linguistic divide for rural Indian farmers. Originally starting as a Telegram bot, the project has evolved into a full-stack platform featuring a real-time web chat widget and a centralized administrative dashboard. The system leverages state-of-the-art Generative AI (Gemini 3.1) coupled with a custom "Nested Multi-Agent Architecture" to provide high-accuracy, multilingual support in Hindi, Bengali, Bhojpuri, and English.

---

### 2. The Brain: Nested Multi-Agent AI Architecture
The core innovation of Agrix lies in its cognitive processing layer, which transitioned from simple prompt-response interactions to a sophisticated multi-agent strategy.

*   **Technology Stack**: Google Gemini 3.1 Flash Lite (Preview) via the `v1beta` API.
*   **The Three-Agent Hierarchy (Implemented in `utils/promptBuilder.js`)**:
    1.  **Strategic Architect**: This agent acts as the system's supervisor. It classifies the user's intent, identifies potential risks (e.g., incorrect pesticide advice), and ensures the overall response aligns with the "Helper of Farmers" persona.
    2.  **Precision Logic Engine**: A "headless" agent that performs calculations and retrieves technical farming data (e.g., soil pH levels, irrigation volumes). It is strictly facts-based and operates without conversational fluff.
    3.  **Interface Concierge**: The final layer responsible for output generation. It translates technical data into empathetic, brief, and clear "layman's terms" in the user's selected language, ensuring accessibility for non-technical users.
*   **Prompt Segregation Policy**: To maintain code purity, the "Prompt Builder" is isolated from the "AI Service." This allows parameters (like layman constraints) to be tweaked without touching the network or retry logic.

---

### 3. Module A: The Real-time Web Chat Widget [NEW EDITION]
The Web Widget brings Agrix to standard browsers, providing a friction-less entry point for users without Telegram.

*   **Communication Protocol**: Socket.IO for duplex, low-latency communication.
*   **The State-Machine Lifecycle**: To prevent unauthorized access and ensure data integrity, the web backend enforces a strict state machine:
    *   `NEW` -> `ONBOARDING` -> `ROUTE_CHOICE` -> `AI_CHAT` -> `CLOSED`.
*   **Interactive Onboarding**: A three-step verification process:
    1.  **Identity Collection**: Capturing full name.
    2.  **Verification**: Validating a 10-digit phone number via regex to ensure data quality.
    3.  **Language Discovery**: A native UI picker allowing users to lock in their language (Bengali, Bhojpuri, etc.) immediately.
*   **Real-time Response Streaming**: Implements a "chunked" delivery system where the AI response is emitted to the frontend in small pieces. This "typing effect" improves perceived performance and keeps users engaged.

---

### 4. Module B: The Unified Admin & Agent Dashboard [NEW EDITION]
The Dashboard serves as the command center for monitoring the platform's health and observing user trends across both platforms.

*   **Backend Aggregation**: The `app.js` file exposes RESTful endpoints (`/api/dashboard/telegram-users` and `/api/dashboard/web-users`) that automatically normalize data from two different MongoDB instances.
*   **Observability Metrics**:
    *   **Telegram Cohort**: Tracks total users, language distribution, usage frequency, and last-seen timestamps.
    *   **Web Cohort**: Specifically monitors active sessions, validated phone numbers, and session duration.
*   **Security & Governance**: 
    *   Namespace-based isolation (Secret keys for the `/widget` namespace vs. protected endpoints).
    *   CORS (Cross-Origin Resource Sharing) white-listing to ensure only authorized web domains can embed the widget.
    *   IP-based rate limiting to prevent DDoS attacks on the AI API.

---

### 5. Backend Engineering: Dual-Database Strategy
To ensure zero interference between platforms, Agrix utilizes a hybrid storage approach:

*   **Database 1 (agrix_telegram)**: Dedicated to the legacy Telegram bot, storing long-term language preferences and user profiles.
*   **Database 2 (agrix_web)**: A highly active database for session tracking, message logging, human-handoff management, and event-sourced idempotency logs.
*   **Logical Isolation**: Implemented in `config/db.js` and `config/webDb.js` using independent Mongoose connection instances.

---

### 6. Critical Technical Features for Academic Review
*   **Idempotency Engine**: Every socket event includes a `messageId` to prevent duplicate processing if a user clicks a button twice or has an unstable network.
*   **Multilingual Mapping**: A centralized `helpers.js` mapping that ensures language consistency across the Telegram bot, the Web interface, and the AI prompt builder.
*   **Graceful Degradation**: If the Gemini API reaches its throughput limit, the system triggers a fallback apology mechanism, ensuring the user is never left with a blank or hanging screen.

---

### 7. Future Directions: Academic Expansion
*   **Voice Integration**: Using STT (Speech-to-Text) for local dialects to aid illiterate farmers.
*   **Image Diagnosis**: Integrating Gemini Vision to analyze crop diseases from agricultural photos.
*   **Contextual CRM**: Integrating the Web Chat history with the Admin Dashboard for better policy-level insights into regional farming crises.

### Conclusion
Agrix represents a paradigm shift in rural communication tools. By moving from a single-channel bot to a multi-platform ecosystem with a dedicated AI architecture, it proves that cutting-edge LLMs can be successfully "tamed" for niche, high-impact societal use cases.
